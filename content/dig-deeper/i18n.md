## Overview

Rok's `rok-i18n` crate provides internationalization with JSON translation files, pluralization support, locale detection, and auto-translated validation messages.

## Translation Files

Translations are stored as JSON files in the `locales/` directory:

```json
// locales/en.json
{
  "welcome": "Welcome back, {name}!",
  "goodbye": "See you later!",
  "post_count": "{count} posts found",
  "post_count_plural": "{count} post found",
  "validation": {
    "required": "The {field} field is required.",
    "email": "Must be a valid email address.",
    "min": "The {field} must be at least {min} characters.",
    "max": "The {field} must not exceed {max} characters."
  },
  "nav": {
    "home": "Home",
    "about": "About Us",
    "contact": "Contact"
  }
}

// locales/es.json
{
  "welcome": "¡Bienvenido de nuevo, {name}!",
  "goodbye": "¡Hasta luego!",
  "post_count": "{count} publicaciones encontradas",
  "post_count_plural": "{count} publicación encontrada",
  "validation": {
    "required": "El campo {field} es obligatorio.",
    "email": "Debe ser una dirección de correo electrónico válida.",
    "min": "El campo {field} debe tener al menos {min} caracteres.",
    "max": "El campo {field} no debe exceder {max} caracteres."
  },
  "nav": {
    "home": "Inicio",
    "about": "Sobre Nosotros",
    "contact": "Contacto"
  }
}
```

## Usage

```rust
use rok_i18n::t;
use rok_i18n::Locale;

// Simple translation
let msg = t!("welcome", name = user.name);
// → "Welcome back, Alice!"

// With count (automatic pluralization)
let msg = t!("post_count", count = posts.len());
// → "3 posts found" (plural)
// → "1 post found" (singular for count = 1)

// Nested keys use dot notation
let msg = t!("validation.required", field = "email");
// → "The email field is required."

// Without interpolation
let msg = t!("goodbye");
// → "See you later!"
```

## Locale Detection

Language is detected from (in order):

1. **Query parameter** — `?lang=es` overrides everything
2. **Cookie** — `locale` cookie for persistent user preference
3. **`Accept-Language` header** — browser preference
4. **Session** — previously selected locale
5. **Default** — fallback locale (typically `en`)

```rust
// Set locale via cookie
use rok_i18n::Locale;

Locale::set("es").await?;

// Get current locale
let current = Locale::current().await?;
```

## Creating Locale Files

```bash
# Scaffold a new locale file from the default locale
rok make:locale es
rok make:locale fr
rok make:locale de
rok make:locale ja
```

## Pluralization Rules

Rok handles pluralization rules for different languages:

- **English**: singular (`count = 1`) vs plural (`count ≠ 1`)
- **French/Spanish**: singular (`count = 0, 1`) vs plural (`count ≥ 2`)
- **Russian/Polish**: complex rules with 3+ forms

The `t!()` macro automatically selects the correct form based on `count`.

## Validation Integration

Validation error messages are automatically translated based on the detected locale:

```rust
#[derive(Validate)]
struct CreateUserRequest {
    #[validate(required, email)]
    email: String,
}

// If locale is "es", errors will be in Spanish:
// "El campo email es obligatorio."
// "Debe ser una dirección de correo electrónico válida."
```

## Configuration

```env
# Default locale
APP_LOCALE=en

# Fallback locale when translation key is missing
APP_FALLBACK_LOCALE=en

# Available locales (comma-separated)
APP_LOCALES=en,es,fr,de,ja
```
