## Overview

Rok's `rok-i18n` crate provides internationalization with JSON translation files.

## Translation Files

Translations are stored in the `locales/` directory as JSON files:

```json
// locales/en.json
{
  "welcome": "Welcome back, {name}!",
  "post_count": "{count} posts found",
  "validation": {
    "required": "The {field} field is required.",
    "email": "Must be a valid email address."
  }
}

// locales/es.json
{
  "welcome": "¡Bienvenido de nuevo, {name}!",
  "post_count": "{count} publicaciones encontradas",
  "validation": {
    "required": "El campo {field} es obligatorio.",
    "email": "Debe ser una dirección de correo electrónico válida."
  }
}
```

## Usage

```rust
use rok_i18n::t;

// Simple translation
let msg = t!("welcome", name = user.name);

// With count
let msg = t!("post_count", count = posts.len());
```

## Locale Detection

Language is detected from:
1. `Accept-Language` header (browser preference)
2. `?lang=` query parameter override
3. Default locale fallback

## Creating Locale Files

```bash
rok make:locale es
```

## Integration

Validation messages are automatically translated based on the detected locale.
